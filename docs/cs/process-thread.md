# 프로세스와 스레드 (Process & Thread)

## 프로세스 (Process)

### 개념
프로세스는 **운영체제로부터 자원을 할당받아 실행 중인 프로그램**이다. 프로그램이 메모리에 올라가 CPU에 의해 실행되는 상태를 의미한다.

### 프로세스의 메모리 구조

```
+------------------+
|      Stack       |  ← 함수 호출, 지역 변수 (높은 주소 → 낮은 주소로 성장)
+------------------+
|        ↓        |
|                  |
|        ↑        |
+------------------+
|       Heap       |  ← 동적 할당 메모리 (낮은 주소 → 높은 주소로 성장)
+------------------+
|      Data        |  ← 전역 변수, 정적 변수 (BSS + Data 세그먼트)
+------------------+
|      Code        |  ← 실행 가능한 코드 (Text 세그먼트)
+------------------+
```

| 영역 | 설명 |
|------|------|
| Code (Text) | 실행할 프로그램 코드 저장 |
| Data | 초기화된 전역/정적 변수 |
| BSS | 초기화되지 않은 전역/정적 변수 |
| Heap | 동적 메모리 할당 영역 (`malloc`, `new`) |
| Stack | 함수 호출 스택, 지역 변수, 반환 주소 |

### 프로세스 상태 (Process State)

```
           fork()
              ↓
           [New]
              ↓ admitted
         [Ready] ←──────────────────┐
              ↓ scheduler dispatch  │
         [Running]                  │ interrupt
              ↓                     │
       ┌──────┴──────┐              │
  I/O wait          exit      time quantum
       ↓                       expired
  [Waiting] ──────────────→ [Ready]
  (Blocked)  I/O complete
```

| 상태 | 설명 |
|------|------|
| New | 프로세스 생성 중 |
| Ready | CPU 할당 대기 중 |
| Running | CPU가 명령을 실행 중 |
| Waiting (Blocked) | I/O 또는 이벤트 완료 대기 중 |
| Terminated | 실행 완료 |

### PCB (Process Control Block)
운영체제가 프로세스를 관리하기 위해 유지하는 자료구조.

- **PID**: 프로세스 고유 식별자
- **프로세스 상태**: Ready, Running, Waiting 등
- **PC (Program Counter)**: 다음 실행할 명령어 주소
- **CPU 레지스터 정보**: 컨텍스트 스위칭 시 저장/복원
- **메모리 관리 정보**: 페이지 테이블, 세그먼트 테이블
- **I/O 상태 정보**: 열린 파일 목록
- **스케줄링 정보**: 우선순위, 스케줄링 큐 포인터

### 컨텍스트 스위칭 (Context Switching)
CPU가 현재 실행 중인 프로세스를 중단하고 다른 프로세스를 실행하는 과정.

```
Process A           OS              Process B
   |                 |                  |
   | running         |                  |
   |── interrupt ──→|                  |
   |  save PCB A    |                  |
   |                |── load PCB B ──→|
   |                |                  | running
   |                |←── interrupt ───|
   |                |  save PCB B      |
   |←── load PCB A ─|                  |
   | running         |                  |
```

> 컨텍스트 스위칭은 **오버헤드**가 발생한다. 스레드 간 전환보다 프로세스 간 전환이 더 비용이 크다.

## 스레드 (Thread)

### 개념
스레드는 **프로세스 내에서 실행되는 흐름의 단위**다. 하나의 프로세스는 여러 스레드를 가질 수 있으며, 각 스레드는 프로세스의 자원을 공유한다.

### 스레드의 메모리 구조

```
+========================+
|       Process          |
|  +------------------+  |
|  |   Code (공유)    |  |
|  +------------------+  |
|  |   Data (공유)    |  |
|  +------------------+  |
|  |   Heap (공유)    |  |
|  +--------+---------+  |
|  |Stack T1|Stack T2 |  |  ← 스레드마다 독립적
|  +--------+---------+  |
+========================+
```

| 공유 자원 | 독립 자원 |
|----------|----------|
| Code 영역 | Stack |
| Data 영역 | PC (Program Counter) |
| Heap 영역 | 레지스터 상태 |
| 열린 파일, 소켓 | Thread ID |

### 멀티 프로세스 vs 멀티 스레드

| 구분 | 멀티 프로세스 | 멀티 스레드 |
|------|------------|------------|
| 메모리 | 독립적 (IPC 필요) | 공유 메모리 |
| 생성 비용 | 높음 | 낮음 |
| 컨텍스트 스위칭 | 느림 | 빠름 |
| 안정성 | 높음 (한 프로세스 죽어도 영향 없음) | 낮음 (한 스레드 오류 → 전체 영향) |
| 통신 | 복잡 (IPC: pipe, socket, shared memory) | 간단 (공유 메모리 직접 접근) |
| 적합한 경우 | 독립된 작업, 안정성 중요 | 병렬 처리, 응답성 중요 |

### 스레드 동기화 문제

#### 경쟁 조건 (Race Condition)
여러 스레드가 공유 자원에 동시에 접근할 때 실행 순서에 따라 결과가 달라지는 현상.

```java
// 문제: 두 스레드가 동시에 실행하면 결과가 예측 불가
int count = 0;

// Thread 1          Thread 2
// read count (0)    read count (0)
// count = 0 + 1     count = 0 + 1
// write count (1)   write count (1)
// → count = 1 (기대값: 2)
```

#### 임계 구역 (Critical Section)
공유 자원에 접근하는 코드 블록. 한 번에 하나의 스레드만 진입해야 한다.

임계 구역 문제 해결의 3가지 조건:
1. **상호 배제 (Mutual Exclusion)**: 한 스레드가 임계 구역에 있으면 다른 스레드는 진입 불가
2. **진행 (Progress)**: 임계 구역에 아무도 없으면 진입 요청한 스레드는 반드시 진입 가능
3. **한정 대기 (Bounded Waiting)**: 스레드가 무한정 대기하지 않도록 보장

#### 동기화 도구

| 도구 | 설명 |
|------|------|
| **Mutex** | 한 번에 하나의 스레드만 잠금 획득 가능. lock/unlock |
| **Semaphore** | 정수 카운터로 여러 스레드 동시 접근 수 제어 |
| **Monitor** | 고수준 동기화 추상화. Java의 `synchronized` |
| **Spinlock** | busy-waiting 방식. 짧은 대기 시간에 적합 |

```java
// Java synchronized 예시
class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;  // 임계 구역 보호
    }
}
```

#### 데드락 (Deadlock)
두 개 이상의 스레드가 서로 상대방이 점유한 자원을 기다리며 무한 대기하는 상태.

**데드락 발생 4가지 조건 (Coffman 조건)**:
1. **상호 배제**: 자원은 한 번에 하나의 스레드만 사용 가능
2. **점유 대기**: 자원을 가진 채로 다른 자원을 기다림
3. **비선점**: 다른 스레드의 자원을 강제로 빼앗지 못함
4. **순환 대기**: 스레드들이 순환 형태로 서로의 자원을 기다림

```
Thread A ──holds──→ Resource 1
   ↑                    |
   |                 waiting
   |                    ↓
Resource 2 ←──holds── Thread B
```

## 프로세스 간 통신 (IPC, Inter-Process Communication)

프로세스는 독립된 메모리 공간을 가지므로 직접 메모리 접근이 불가하다.

| 방법 | 설명 | 특징 |
|------|------|------|
| **Pipe** | 단방향 통신. 부모-자식 프로세스 간 | 단순, 단방향 |
| **Named Pipe (FIFO)** | 파일 시스템 기반 양방향 파이프 | 이름 있는 파이프 |
| **Message Queue** | 메시지를 큐에 저장하여 전달 | 비동기, 구조화된 데이터 |
| **Shared Memory** | 메모리 공간을 직접 공유 | 가장 빠름, 동기화 필요 |
| **Socket** | 네트워크 기반 통신 | 다른 머신 간에도 사용 가능 |
| **Signal** | 프로세스에 이벤트 알림 | 단순 알림 용도 |

## 면접 빈출 질문

**Q. 프로세스와 스레드의 차이는?**
> 프로세스는 독립된 메모리 공간을 가진 실행 단위이고, 스레드는 프로세스 내에서 메모리를 공유하는 실행 흐름이다. 스레드는 생성 비용이 낮고 통신이 쉽지만, 한 스레드의 오류가 전체 프로세스에 영향을 줄 수 있다.

**Q. 컨텍스트 스위칭이란?**
> CPU가 현재 실행 중인 프로세스/스레드의 상태(PCB)를 저장하고 다른 프로세스/스레드의 상태를 복원하여 실행을 전환하는 과정이다. 오버헤드가 발생하며, 프로세스 간 전환이 스레드 간 전환보다 비용이 더 크다.

**Q. 데드락을 방지하는 방법은?**
> Coffman의 4가지 조건 중 하나를 제거하면 된다. 예를 들어 자원 할당 순서를 일정하게 정해 순환 대기를 방지하거나, 자원 요청 시 모든 자원을 한 번에 요청하는 방식으로 점유 대기를 방지할 수 있다.

**Q. 멀티 스레드 환경에서 동기화가 필요한 이유는?**
> 여러 스레드가 공유 메모리에 동시에 접근하면 경쟁 조건이 발생하여 데이터 일관성이 깨질 수 있기 때문이다. Mutex, Semaphore 등의 동기화 도구로 임계 구역을 보호해야 한다.
